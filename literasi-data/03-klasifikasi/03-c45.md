# Algoritma C4.5 (Decision Tree)

**Kategori:** Klasifikasi | **Level:** Menengah

## Ringkasan
C4.5 (Quinlan, 1993) membangun pohon keputusan dengan membagi data berdasar fitur yang memberikan information gain ratio tertinggi. Mendukung fitur kontinu dan missing values. Keunggulan utama: hasil dapat dibaca manusia.

## Algoritma
1. Hitung entropi data.
2. Untuk setiap fitur, hitung gain ratio jika data dibagi menurut fitur itu.
3. Pilih fitur dengan gain ratio tertinggi sebagai node split.
4. Ulangi rekursif hingga daun murni atau kriteria berhenti.
5. Pruning untuk mengurangi overfitting.

## Rumus
- **Entropi:** $H(S) = -\sum_{i=1}^{c} p_i \log_2 p_i$
- **Information Gain:** $IG(S, A) = H(S) - \sum_{v \in A} \frac{|S_v|}{|S|} H(S_v)$
- **Gain Ratio:** $GR(A) = IG(A) / SplitInfo(A)$

## Studi Kasus PANDORA
Klasifikasi pegawai "Anomali / Normal" berdasar present_rekap. Pohon yang terbentuk:

```
jarak_geofence > 500m?
 ├── Ya → velocity > 200 km/jam?
 │        ├── Ya → ANOMALI (fake GPS)
 │        └── Tidak → NORMAL (dinas luar)
 └── Tidak → IMEI_match?
          ├── Tidak → ANOMALI (device tidak terdaftar)
          └── Ya → NORMAL
```

Keunggulan: aturan ini dapat dibaca dan dijelaskan ke atasan/inspektorat.

```python
from sklearn.tree import DecisionTreeClassifier, export_text
tree = DecisionTreeClassifier(criterion='entropy', max_depth=4)
tree.fit(X_train[['jarak_geofence','velocity','imei_match']], y_train)
print(export_text(tree, feature_names=['jarak_geofence','velocity','imei_match']))
```

## Pitfalls
- Pohon dalam = overfit. Batasi max_depth atau lakukan pruning.
- Sensitif terhadap variasi kecil data → gunakan Random Forest untuk stabilitas.
- sklearn memakai CART, bukan C4.5 murni; untuk C4.5 gunakan Weka atau chefboost.
