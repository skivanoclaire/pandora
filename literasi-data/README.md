# Literasi Data

Modul pembelajaran data science terstruktur, dirancang khusus untuk lingkungan PANDORA (Sistem Kehadiran Pegawai Pemerintah Provinsi Kalimantan Utara). Setiap konsep dijelaskan secara ringkas dan, jika memungkinkan, dikaitkan langsung dengan data nyata pada PANDORA (6.475 pegawai aktif, log presensi, data OPD).

## Cara Penggunaan

Modul ini terdiri dari **7 kategori**. Klik kategori untuk melihat daftar konsep di dalamnya, lalu klik konsep untuk membaca penjelasan lengkap.

## Daftar Kategori

### [1. Fondasi](01-fondasi/README.md)
Konsep dasar yang harus dipahami sebelum masuk ke teknik analitik lanjutan. *(7 konsep)*

### [2. Data Engineering](02-data-engineering/README.md)
Pengelolaan, persiapan, dan transformasi data sebelum dianalisis. *(11 konsep)*

### [3. Klasifikasi](03-klasifikasi/README.md)
Algoritma untuk memprediksi label/kategori dan cara mengevaluasinya. *(13 konsep)*

### [4. Estimasi & Regresi](04-estimasi-regresi/README.md)
Algoritma untuk memprediksi nilai numerik dan metrik evaluasinya. *(8 konsep)*

### [5. Clustering](05-clustering/README.md)
Pengelompokan data tanpa label dan metrik evaluasi clustering. *(7 konsep)*

### [6. Association Rule](06-association-rule/README.md)
Mencari pola hubungan antar item dalam data transaksi. *(2 konsep)*

### [7. Data Tak Terstruktur](07-data-tak-terstruktur/README.md)
Pengolahan teks (NLP) dan citra (Computer Vision). *(9 konsep)*

---

## Saran Jalur Belajar

**Pemula:** Fondasi → Data Engineering → Klasifikasi → Performance Klasifikasi
**Menengah:** + Estimasi & Regresi → Clustering
**Lanjut:** + Association Rule → Data Tak Terstruktur (NLP/CV)

## Konvensi Penulisan

Setiap halaman konsep menggunakan struktur seragam: Ringkasan, Penjelasan, Rumus/Algoritma, Studi Kasus PANDORA atau Contoh, Pitfalls, dan Kaitan dengan topik lain.
